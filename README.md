# Dublin Derelict Sites Register

An open re-publication and visualisation of Dublin City Council's Derelict
Sites Register. The register is public data, but the council publishes it
through an ArcGIS Experience Builder application that makes the underlying
data awkward to access. This project retrieves the data from the public
feature service twice daily and republishes it in open formats, alongside a
map, table, and some light spatial analysis.

## Data

- `data/derelict_sites_register.geojson` - all sites as GeoJSON points (WGS84)
- `data/derelict_sites_register.csv` - the attribute table as CSV
- `data/active_cases_grid.geojson` - the council's grid aggregation of all
  active dereliction cases, a considerably larger set than the published
  register. Note: despite the source layer's name, the cells are about
  600 m across on the ground, not 1000 m - the grid was generated in Web
  Mercator, whose nominal metres are inflated by 1/cos(latitude) at Dublin

These files are regenerated by `scripts/fetch_data.py`, which queries the
public ArcGIS feature service and writes deterministic output (features
ordered by OBJECTID, sorted keys) so that diffs only appear when the register
itself changes.

The Vacant Sites Register is published separately:

- `data/vacant_sites_register.geojson` - the register as polygon boundaries
  (WGS84), with register number, address, ownership, folio reference,
  valuation, status, and date registered. Each site also carries the planning
  applications recorded against it (newest first), with the application type,
  decision outcome, registration date, and a link to the council's planning
  portal
- `data/vacant_sites_register.csv` - the site attribute table as CSV
- `data/vacant_sites_planning_links.csv` - one row per site-to-application link
- `data/vacant_sites_enrichment.csv` - ownership, folio, and valuation fields
  from the register PDF published on dublincity.ie (currently the 25/08/2025
  edition), extracted by `scripts/extract_vacant_sites_pdf.py`
- `data/vacant_sites_removed.csv` - sites that appeared on Smart Dublin's
  12/04/2023 register snapshot but have since been removed from the register

These are regenerated by `scripts/fetch_vacant_sites.py`. Unlike the Derelict
Sites Register, the Vacant Sites Register is published only through DCC's
MapZone planning viewer, whose backing server exposes a SQL-query endpoint
returning GeoJSON. The script authenticates with the public account embedded
in the viewer (no secret required), so it also runs in CI.

The MapZone feed's ownership and valuation attributes are stale: most market
values are missing and several owner names are superseded (verified against
the register PDF, whose ownership matches the independent 2023 snapshot where
the feed disagrees). The fetch script therefore replaces the feed's ownership,
owner address, folio reference, and valuation with the PDF values at build
time, and adds the PDF's valuation date. Register membership, geometry,
status, and registration dates still come from the live feed, which tracks
removals the static PDF cannot. When DCC publishes a new register PDF, rerun
`extract_vacant_sites_pdf.py` against it to refresh the enrichment file.

The Vacant Sites Register is historical: the vacant sites levy was replaced
by the Residential Zoned Land Tax (RZLT) and the register has received no new
entries since April 2022. Its living successor is published as part of the
RZLT final map, an ArcGIS feature service operated by the Department of
Housing, Local Government and Heritage:

- `data/rzlt_sites.geojson` - Dublin City parcels from the RZLT final map
  whose inclusion required a vacant-or-idle determination: the mixed-use
  generalised zoning types (M1, M2, M3) and Dublin City's R3 parcels (Z14
  strategic development and regeneration areas). Blanket residential
  zonings, which are included for being serviced rather than vacant, are
  excluded
- `data/rzlt_sites.csv` - the parcel attribute table as CSV
- `data/rzlt_vacant_links.csv` - parcels matched to the Vacant Sites
  Register site they continue (DCC seeded its RZLT submission from the
  register), built spatially by `scripts/link_rzlt_vacant_sites.py`
  (requires shapely, run manually) and merged into the parcel attributes

These are regenerated by `scripts/fetch_rzlt.py`, which runs alongside the
other fetch scripts in CI.

The council does not publish a site-to-application list, so each link is
reconstructed from two independent signals that must agree: the application
footprint overlaps the site boundary, *and* the application appears in the
council's own related-applications record for the site's best-overlapping
application. Requiring both discards footprints that merely touch the boundary
and the very large related-application clusters the council keeps for
regeneration areas. Two sites have no strongly-overlapping application to
anchor the cross-check and fall back to the spatial signal alone; their links
are flagged `council_confirmed = false`.

A GitHub Actions workflow runs both scripts at 00:01 and 12:01 UTC and commits
the result only when the data has changed.

Sources: Dublin City Council, DCC Derelict Sites Register Public points view
(ArcGIS Online feature service); DCC Vacant Sites Register (MapZone planning
viewer; ownership and valuations from the register PDF on dublincity.ie;
removed sites from Smart Dublin's April 2023 register snapshot); Residential
Zoned Land Tax final map, copyright Department of Housing, Local Government

## Site

The visualisation is a static page served from GitHub Pages, with no build
step. MapLibre GL renders the map (points coloured by time on the register,
with optional hexagonal density, Voronoi catchment, and protected-structure
emphasis overlays), D3 renders the charts, and Turf.js does the client-side
spatial analysis: hexagonal binning, Voronoi tessellation, distance from the
city centre, and a Clark-Evans nearest-neighbour clustering index.
The points are also loaded into an RBush spatial index, which powers the
"closest derelict site" geolocation lookup.

Problems with the site or the data can be reported through the issue
templates: https://github.com/urschrei/derelict_sites_register/issues/new/choose

## Development

```sh
python3 scripts/fetch_data.py   # refresh data locally
python3 -m http.server          # serve the site at http://localhost:8000
```
