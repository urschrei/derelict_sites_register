# Dublin Derelict Sites Register

An open re-publication and visualisation of Dublin City Council's Derelict
Sites Register. The register is public data, but the council publishes it
through an ArcGIS Experience Builder application that makes the underlying
data awkward to access. This project retrieves the data from the public
feature service twice daily and republishes it in open formats, alongside a
map, table, and some light spatial analysis.

## Data

- `data/derelict_sites_register.geojson` - all sites as GeoJSON points (WGS84)
- `data/derelict_sites_register.csv` - the attribute table as CSV
- `data/active_cases_grid.geojson` - the council's grid aggregation of all
  active dereliction cases, a considerably larger set than the published
  register. Note: despite the source layer's name, the cells are about
  600 m across on the ground, not 1000 m - the grid was generated in Web
  Mercator, whose nominal metres are inflated by 1/cos(latitude) at Dublin

These files are regenerated by `scripts/fetch_data.py`, which queries the
public ArcGIS feature service and writes deterministic output (features
ordered by OBJECTID, sorted keys) so that diffs only appear when the register
itself changes.

The Vacant Sites Register is published separately:

- `data/vacant_sites_register.geojson` - the register as polygon boundaries
  (WGS84), with register number, address, ownership, folio reference,
  valuation, status, and date registered
- `data/vacant_sites_register.csv` - the same attribute table as CSV
- `data/vacant_sites_planning_links.csv` - the planning applications whose
  footprint falls within each site boundary. The two registers share no key,
  so the link is spatial; there are around 300 such applications across the 33
  sites

These are regenerated by `scripts/fetch_vacant_sites.py`. Unlike the Derelict
Sites Register, the Vacant Sites Register is published only through DCC's
MapZone planning viewer, whose backing server exposes a SQL-query endpoint
returning GeoJSON. The script authenticates with the public account embedded
in the viewer (no secret required), so it also runs in CI.

A GitHub Actions workflow runs both scripts at 00:01 and 12:01 UTC and commits
the result only when the data has changed.

Sources: Dublin City Council, DCC Derelict Sites Register Public points view
(ArcGIS Online feature service); DCC Vacant Sites Register (MapZone planning
viewer).

## Site

The visualisation is a static page served from GitHub Pages, with no build
step. MapLibre GL renders the map (points coloured by time on the register,
with optional hexagonal density, Voronoi catchment, and protected-structure
emphasis overlays), D3 renders the charts, and Turf.js does the client-side
spatial analysis: hexagonal binning, Voronoi tessellation, distance from the
city centre, and a Clark-Evans nearest-neighbour clustering index.
The points are also loaded into an RBush spatial index, which powers the
"closest derelict site" geolocation lookup.

Problems with the site or the data can be reported through the issue
templates: https://github.com/urschrei/derelict_sites_register/issues/new/choose

## Development

```sh
python3 scripts/fetch_data.py   # refresh data locally
python3 -m http.server          # serve the site at http://localhost:8000
```
