# Dublin Derelict Sites Register

An open re-publication and visualisation of Dublin City Council's Derelict
Sites Register. The register is public data, but the council publishes it
through an ArcGIS Experience Builder application that makes the underlying
data awkward to access. This project retrieves the data from the public
feature service twice daily and republishes it in open formats, alongside a
map, table, and some light spatial analysis.

## Data

- `data/derelict_sites_register.geojson` - all sites as GeoJSON points (WGS84)
- `data/derelict_sites_register.csv` - the attribute table as CSV

Both files are regenerated by `scripts/fetch_data.py`, which queries the
public ArcGIS feature service and writes deterministic output (features
ordered by OBJECTID, sorted keys) so that diffs only appear when the register
itself changes. A GitHub Actions workflow runs the script at 00:01 and 12:01
UTC and commits the result only when the data has changed.

Source: Dublin City Council, DCC Derelict Sites Register Public points view
(ArcGIS Online feature service).

## Site

The visualisation is a static page served from GitHub Pages, with no build
step. MapLibre GL renders the map (points coloured by time on the register,
with optional hexagonal density, Voronoi catchment, and protected-structure
emphasis overlays), D3 renders the charts, and Turf.js does the client-side
spatial analysis: hexagonal binning, Voronoi tessellation, distance from the
city centre, and a Clark-Evans nearest-neighbour clustering index.
The points are also loaded into an RBush spatial index, which powers the
"closest derelict site" geolocation lookup.

Problems with the site or the data can be reported through the issue
templates: https://github.com/urschrei/derelict_sites_register/issues/new/choose

## Development

```sh
python3 scripts/fetch_data.py   # refresh data locally
python3 -m http.server          # serve the site at http://localhost:8000
```
